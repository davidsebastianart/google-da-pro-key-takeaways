## Key Takeaways (Differentiate data formats and structures)

- Data types
	- **Qualitative Data**: This includes non-numeric information such as movie titles and cast members, which cannot be measured or counted.
	- **Quantitative Data**: This can be measured or counted, represented by numbers, such as a movie's budget and box office revenue.

- Data categories
	- **Discrete Data**: Countable data with a limited number of values, like budget and box office returns.
	- **Continuous Data**: Data is measured and can have almost any numeric value, such as a movie's runtime.

- Data Structure
	- **Nominal Data**: Qualitative data without a specific order, like responses to whether someone has watched a movie.
	- **Ordinal Data**: Qualitative data with a defined order, such as ranking a movie on a scale from 1 to 5.

- Data Sources
	- **Internal Data**: Data generated within an organization, often more reliable.
	- **External Data**: Data sourced from outside an organization, valuable for comprehensive analysis.

- Data Organization
	- **Structured Data**: Organized in a specific format (e.g., spreadsheets), making it easy to search and analyze.
	- **Unstructured Data**: Not organized in a clear manner, such as audio and video files, which can complicate analysis.

- **Primary Data**: Collected by a researcher from first-hand sources (e.g., interviews, surveys).
- **Secondary Data**: Gathered by other people or from other research (e.g., census data, purchased datasets). 

- **Data Model**: a model that is used for organizing data elements and how they relate to one another.

- **Data Elements**: pieces of information, such as people's names, account numbers, and addresses.

- **Data modeling** is the process of creating diagrams that visually represent how data is organized and structured, similar to a blueprint for a house. Different users may have varying data needs, but a data model provides a comprehensive understanding of the overall data structure. [[Data modeling levels and techniques]]
	
	- **Levels of Data Modeling**
		- **Conceptual Data Modeling:** Offers a high-level view of data interactions within an organization, focusing on business requirements without technical details.
		- **Logical Data Modeling:** Details the technical aspects, such as relationships and attributes, without specifying actual database table names.
		- **Physical Data Modeling:** Depicts how a database operates, including specific entities, attributes, table names, and data types.
	
	- **Data Modeling Techniques**
		Common methods include the **Entity Relationship Diagram (ERD)** for visualizing relationships between entities and the **Unified Modeling Language (UML) diagram** for detailed system structure.

## Key Takeaways (Explore data types, fields, and values)

- **Data type**: a specific kind of data attribute that tells what kind of values the data is.
	- Data types in spreadsheets:
	  - Number
	  - Text or string
	  - Boolean [[Use Boolean logic]]

- **Data Table Structure**
	- Data tables are organized in **rows (records)** and **columns (fields)**.
	- Each record contains the same fields in the same order, representing consistent information.

- **Understanding Wide and Long Data**
	- **Wide data** has each subject represented in a single row with multiple columns for various attributes, making it useful for comparing specific attributes across subjects.
	- **Long data** has each subject represented by multiple rows, allowing for comparisons over time or across subjects.

- **Understanding Data Transformation** [[Transforming data]]
	- Data transformation involves changing the format, structure, or values of data to enhance its usability.
	- Common transformation tasks include adding or deleting data, standardizing variable names, and saving files in different formats.
	
	- **Reasons for Data Transformation**
		- Goals include better data organization, compatibility across systems, and enhanced data comparison.
		- For example, merging customer databases from different companies requires transforming data formats to ensure compatibility.
